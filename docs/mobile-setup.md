# Running Ubapp on a separate phone

This guide covers installing Ubapp onto a physical Android or iOS phone, the dependencies you need on your dev machine and on the phone, and how to run a multi-device session (host phone + one or more peer phones / browser guests).

The repo ships Dart source only — `ios/` and `android/` are generated by `flutter create` on first checkout. Everything below assumes you've already cloned the repo.

---

## 1. Dev-machine prerequisites

### Both platforms

| Tool | Version | Notes |
|---|---|---|
| Flutter SDK | ≥ 3.27 | `flutter --version` to check |
| Dart SDK | ≥ 3.6 (bundled with Flutter) | |
| Git | any recent | |

Verify your toolchain end-to-end:

```bash
flutter doctor
```

Resolve every red ✗ for the platform(s) you intend to ship to before continuing.

### Android-specific

- **Android Studio** (for the Android SDK + platform-tools / `adb`). Command-line tools alone work but Studio is the easiest path.
- **Android SDK platform 34+** and **build-tools 34+** (BLE peripheral APIs need `compileSdk ≥ 31`, but newer is fine).
- **USB cable** *or* same Wi-Fi network for wireless `adb`.
- Phone with **Developer Options + USB debugging** enabled.

### iOS-specific

- **macOS** (required — iOS builds cannot be produced on Linux/Windows).
- **Xcode 15+** with the iOS 17 SDK installed (`xcode-select --install` for command-line tools, then open Xcode once to accept the licence).
- **CocoaPods** ≥ 1.13: `sudo gem install cocoapods`.
- An **Apple ID** signed into Xcode (Settings → Accounts). A free personal team is enough for sideloading to your own device for 7 days; a paid Apple Developer account lifts that to 1 year.
- Phone running **iOS 13+** with **Developer Mode** enabled (Settings → Privacy & Security → Developer Mode on iOS 16+).
- A **Lightning / USB-C cable** for first install (after that, wireless debugging works once the device is paired in Xcode).

## 2. Flutter package dependencies

These are declared in `pubspec.yaml` and pulled in by `flutter pub get`. You shouldn't have to edit any of them — listed here so you know what's running on the phone:

| Package | Used for |
|---|---|
| `flame` | Real-time demo (`lib/realtime/`) |
| `flutter_blue_plus` | BLE **central** scanning for tag proximity |
| `permission_handler` | Runtime Bluetooth / location prompts on Android & iOS |
| `network_info_plus` | Reads the phone's Wi-Fi IP for the host's QR / join URL |
| `qr_flutter` | Renders the join-URL QR code in the host UI |
| `shelf`, `shelf_web_socket`, `web_socket_channel` | Host HTTP + WebSocket server for browser-tier games |
| `vibration` | Role-change haptics in tag |

BLE **peripheral** advertising is not on pub — it's wired through a `MethodChannel` to native code that you copy in once (see step 5 below).

## 3. First-time project setup

From the repo root, on the dev machine:

```bash
# Generate ios/ and android/ shells — only needed once per checkout.
flutter create . --project-name ubapp --org com.example --platforms=ios,android

# Pull dependencies declared in pubspec.yaml.
flutter pub get

# Sanity-check the toolchain and the generated shells.
flutter doctor
flutter analyze
```

## 4. Add the required permissions

`flutter create` produces empty manifests. Paste the snippets below into the generated files before your first run, or BLE and local-network access will be silently denied at runtime.

### Android — `android/app/src/main/AndroidManifest.xml`

Inside the top-level `<manifest>` element, above `<application>`:

```xml
<uses-permission android:name="android.permission.BLUETOOTH_SCAN"
    android:usesPermissionFlags="neverForLocation" />
<uses-permission android:name="android.permission.BLUETOOTH_ADVERTISE" />
<uses-permission android:name="android.permission.BLUETOOTH_CONNECT" />
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
<uses-permission android:name="android.permission.ACCESS_WIFI_STATE" />

<!-- Pre-Android 12 fallback. -->
<uses-permission android:name="android.permission.BLUETOOTH"
    android:maxSdkVersion="30" />
<uses-permission android:name="android.permission.BLUETOOTH_ADMIN"
    android:maxSdkVersion="30" />
```

In `android/app/build.gradle` (or `build.gradle.kts`) make sure `compileSdk` is **34 or higher** and `minSdk` is **21 or higher**.

### iOS — `ios/Runner/Info.plist`

Inside the top-level `<dict>`:

```xml
<key>NSBluetoothAlwaysUsageDescription</key>
<string>Used to find nearby phones for tag and other party games.</string>
<key>NSLocalNetworkUsageDescription</key>
<string>Used to host or join games on the local Wi-Fi.</string>
<key>NSBonjourServices</key>
<array><string>_ubapp._tcp</string></array>
```

## 5. (Tag only) Install the native BLE advertiser

## 6. Running on the phone

Connect the phone, then list devices:

```bash
flutter devices
```

You should see your phone listed alongside any emulators. Pick one and launch:

```bash
# Run on the first connected device.
flutter run

# Or target a specific device by id from `flutter devices`.
flutter run -d <device-id>
```

### Android — first run specifics

1. Plug the phone in over USB. Accept the **"Allow USB debugging?"** prompt on the phone (tick *Always allow from this computer*).
2. `flutter run` will install the debug APK and stream logs over `adb`.
3. To go wireless after the initial pair: `adb tcpip 5555 && adb connect <phone-ip>:5555`.

### iOS — first run specifics

1. Open `ios/Runner.xcworkspace` in Xcode once. Select the **Runner** target → **Signing & Capabilities** → set **Team** to your Apple ID. This populates the bundle identifier and provisioning profile.
2. Plug the phone in over USB. Trust the computer on the phone when prompted.
3. Back on the CLI: `flutter run`. Xcode will build, sign, and install the app.
4. On the phone, the first launch may fail with *"Untrusted Developer"* — go to **Settings → General → VPN & Device Management**, tap your Apple ID, and choose *Trust*. Re-launch the app.
5. For wireless runs afterwards, tick **Connect via network** for the device in Xcode's *Devices and Simulators* window.

## 7. Multi-device play

All cross-device modes assume **both phones are on the same Wi-Fi network** (the host server binds to the LAN IP and the BLE advertiser needs phones in physical proximity).

### Browser-tier games (Mafia / Imposter / Crazy Eights)

You only need **one phone with the app installed**; everyone else joins via any browser.

1. On the host phone, open the game and tap **Host**.
2. The screen shows a QR code and a `http://<wifi-ip>:7654/` URL.
3. Other players scan the QR with their phone camera (any modern browser will open the URL) — no app install needed on guest phones.
4. Once everyone's joined, the host taps **Start**.

If guests can't load the page, the host phone's Wi-Fi is probably on a network with **client isolation** enabled (common on guest / public Wi-Fi). Use a personal hotspot or a home network instead.

### Tag (BLE proximity, needs the app on every phone)

1. Install the app on **every** participating phone (steps 3–6 above).
2. On each phone, grant **Bluetooth** + **Nearby devices** / **Local network** permissions when prompted.
3. Host phone: open **Tag → Host a game**, toggle *Use real BLE* on, share the QR / URL.
4. Each peer phone: open **Tag → Join**, paste the host URL (or scan the QR), tap **Join**.
5. Host confirms the roster → **Start**. Both phones now run the same `TagEngine` against the same ordered events.

iOS-to-iOS tag requires the native BLE advertiser from step 5; Android-only or mixed iOS-Android also need both halves of `tooling/ble_native/` installed in their respective shells.

## 8. Troubleshooting

| Symptom | Likely cause |
|---|---|
| `flutter devices` doesn't show your phone | USB debugging off (Android), or computer not trusted on the phone (iOS) |
| iOS install fails with *"Could not find Developer Disk Image"* | Xcode version is older than the iOS version on the phone — update Xcode |
| Android install fails with `INSTALL_FAILED_USER_RESTRICTED` | "Install via USB" blocked in Developer Options; toggle it on |
| QR code loads in the browser but WebSocket never connects | Wi-Fi has client isolation, or a VPN is intercepting LAN traffic |
| BLE toggle in tag lobby stays disabled | Native advertiser plugin not installed — re-do step 5 |
| `NSLocalNetworkUsageDescription` not prompting on iOS | Key missing from `Info.plist`; iOS caches the denial — delete the app and reinstall |
| `flutter pub get` fails after `flutter create` | Old Flutter SDK; run `flutter upgrade` and retry |

If something else breaks, run `flutter run -v` and search the verbose output for the first red `Error:` line — it almost always names the missing permission or unsigned framework.
